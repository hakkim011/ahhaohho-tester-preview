## 1. 개발자 작업 — User Property 태깅

체험단(크루) 어드민에서 참여자 등록 / 상태 변경 시 서버가 Amplitude HTTP Identify API를 자동 호출하여 아래 user property를 설정.

**설정할 User Properties**

| Property | 타입 | 값 예시 | 설명 |
|----------|------|---------|------|
| `is_crew` | Boolean | `true` / `false` | 크루 참여 이력 — 한 번 참여하면 계속 `true` 유지 |
| `crew_status` | String | `active` / `dropped` / `completed` | 현재 상태 (참여 중 / 탈락 / 수료) |
| `crew_program` | String | `akk` | 프로그램 구분 (아크크) |
| `crew_cohort` | String | `akk-1` | 현재(최근) 기수 — 문자열로 확장성 확보 |
| `crew_cohort_count` | Int | `2` | 참여 기수 수 (중복 참여 시 누적) |
| `crew_cohort_history` | [String] | `["akk-1", "akk-2"]` | 참여 기수 이력 (중복 참여 시 누적) |

**API 호출 예시 (Server-side / HTTP)**

```
POST https://api2.amplitude.com/identify
Content-Type: application/x-www-form-urlencoded

api_key=<API_KEY>&identification=[{
  "user_id": "<user_id>",
  "user_properties": {
    "$set": {
      "is_crew": true,
      "crew_status": "active",
      "crew_program": "akk",
      "crew_cohort": "akk-1",
      "crew_cohort_count": 1,
      "crew_cohort_history": ["akk-1"]
    }
  }
}]
```

**설정 시점**

- 어드민에서 참여자 등록 시 → `is_crew: true`, `crew_status: "active"`, `crew_program`, `crew_cohort`, `crew_cohort_count`, `crew_cohort_history` 설정
- 어드민에서 탈락 처리 시 → `crew_status: "dropped"` (단, `is_crew`는 `true` 유지 — 이력 보존)
- 기수 수료 시 → `crew_status: "completed"`
- 기수 종료 후에도 `is_crew: true` 유지 (참여 이력·배지 보존) — 상태는 `crew_status`로 구분
- 중도 참가(기수 도중 합류) 시 → 등록 즉시 위와 동일하게 설정 (동일 기수 기준 적용)
- 중복 참여(이전 기수 이력 보유자가 신규 기수 등록) 시 → `crew_cohort`를 신규 기수로 갱신, `crew_cohort_count` +1, `crew_cohort_history`에 신규 기수 추가

## 2. Amplitude 코호트 생성 — 운영자 작업

태깅 완료 후 Amplitude에서 코호트 생성.

**전체 크루 코호트**

```
Audiences → Cohorts → Create Cohort
조건: User Property / is_crew / = / true
```

**기수별 코호트**

```
조건: User Property / is_crew / = / true
  AND User Property / crew_cohort / = / akk-1
```

**활동 중인 크루만** (탈락 제외)

```
조건: User Property / is_crew / = / true
  AND User Property / crew_status / is / active, completed
```

---

## 3. 활용

- **Spot check**: Amplitude 분석 필터에 코호트 적용 → 체험단 계정 활동만 추려서 확인
- **기수별 비교**: `crew_cohort` 값(`akk-1`, `akk-2`…)으로 기수별 행동 패턴 비교 가능
- **상태별 분석**: `crew_status`로 참여 중(active)·탈락(dropped)·수료(completed) 세분화 가능
- **전환율 추적**: 체험단 종료 후 일반 유저로 전환 여부 모니터링
