# 체험단 배지 시스템

> 작성자: 김하경(하디)
> 작성일: 2026-06-16

---

## 개요

체험단 참여자에게 기수 시작일에 배지를 부여한다.
배지는 위치에 따라 노출 기간이 다르게 적용된다.
프로필에는 영구적으로 새겨지고, 갤러리·댓글에는 기수 종료일 이후 30일까지 표시된다.

---

## 노출 정책

| 노출 위치 | 기간 | 비고 |
|-----------|------|------|
| 프로필 화면 | **영구** | 수료 기념, 항상 표시 |
| 갤러리 게시물 카드 (작성자 닉네임 옆) | **체험단 종료일 이후 30일** | 기간 종료 시 자동 비노출 |
| 댓글 (닉네임 옆) | **체험단 종료일 이후 30일** | 기간 종료 시 자동 비노출 |

> 비활동 유저는 게시·댓글 활동이 없으면 갤러리·댓글 배지 노출 자체가 발생하지 않으므로 별도 처리 불필요

---

## 부여 조건

### 부여 대상
- 해당 기수 체험단 참여자 전원

### 부여 시점
- **기수 시작일** (오리엔테이션 당일)에 일괄 부여
- 어드민에서 참여자를 기수에 등록하는 시점에 자동 부여
- 중도 참가자도 동일. 배지 노출 기간은 항상 **기수의 시작일·종료일 기준**으로 계산되며, 개인 등록일에 따라 조정되지 않음

### 부여 주체
- 운영자 (어드민 체험단 관리 화면에서 처리)

---

## 예외 처리

| 상황 | 처리 방식 |
|------|-----------|
| 중도 이탈 (자진) | 배지 유지. 프로필 영구, 갤러리·댓글은 기수 종료일 기준 30일까지 동일 적용 |
| **탈락 처리** (운영자 조치) | `is_tester: false` 처리 → 모든 위치(프로필·갤러리·댓글)에서 즉시 비노출 |
| 탈락 후 재등록 | 별도 복귀 기능 없음. 어드민 "참여자 추가"로 재등록 시 배지 재부여되며 기수 기준 노출 기간 동일 적용 |
| 참여 중 탈퇴 | 계정과 함께 소멸 (복구 불가) |

---

## 운영 절차

```
기수 시작일 (오리엔테이션 당일)
  → 어드민: 참여자 등록 → 배지 자동 부여
  → 단톡 공지: "배지 부여 완료" 안내

운영 중 탈락 발생 시
  → 어드민: 해당 참여자 '탈락' 처리
  → 배지 전 위치(프로필·갤러리·댓글) 즉시 비노출
  → 인앱 알림 발송 없음

기수 종료일 + 30일
  → 갤러리·댓글 배지 자동 비노출 (별도 운영 조치 불필요)
  → 프로필 배지는 계속 유지
```

---

## 노출 조건 상세

**프로필**
`is_tester = true`

**갤러리·댓글**
`is_tester = true` AND `오늘 ≤ cohort[tester_cohort].end_date + 30일`
- `tester_cohort` 값으로 어드민 기수 테이블의 종료일을 조회해 계산
- 별도 user property 추가 없이 기존 구조 그대로 활용

---

## 개발 요건 요약

| 요건 | 내용 |
|------|------|
| 어드민 참여자 등록 시 배지 자동 부여 | 기수에 참여자 추가 → `is_tester: true`, `tester_cohort: n` 설정 |
| 어드민 탈락 처리 시 배지 즉시 비노출 | `is_tester: false` 처리 → 프로필·갤러리·댓글 모든 위치 자동 비노출 |
| 갤러리·댓글 노출 기간 자동 만료 | 앱이 `tester_cohort`로 기수 종료일 조회 → `오늘 > 종료일 + 30일`이면 비노출 |
| 프로필 배지 표시 영역 | 디자인 확정 후 개발 |
| 갤러리·댓글 배지 아이콘 | 닉네임 옆 소형 아이콘, 조건부 노출 |

| 탈퇴 계정 처리 | 배지 소멸 여부 정책 확정 |
