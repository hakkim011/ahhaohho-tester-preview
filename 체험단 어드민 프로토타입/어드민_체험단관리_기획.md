## 기획 배경

아하오호 아크크 체험단 운영(2026년 8월 초 시작)을 앞두고 참여자 관리 및 데이터 분석을 위한 내부 도구가 필요함

## 기획 의도

아하오호 아크크 체험단을 운영하면서 참여자를 체계적으로 관리하고, 체험단 계정을 일반 유저와 구분하여 Amplitude에서 별도로 분석할 수 있도록 하기 위함. 참여자 등록 시 자동으로 user property가 태깅되어 운영 지표를 정확하게 측정할 수 있고, 기수별 현황을 한눈에 파악할 수 있도록 함

## 기능 요약

### 기수 관리

- 기수 추가/수정 (운영 기간, 메모) — 수정 시 기수 번호는 변경 불가, 운영 기간·메모만 편집 가능
- 운영 상태 자동 계산 — 날짜 기준으로 운영 전 / 운영 중 / 종료 자동 태깅 (수동 설정 없음)
- 기수별 참여자 수 표시
- 복수 기수 동시 운영 가능 (기수 간 기간 중복 허용)
- 상단 통계 카드
  - 현재 운영 중인 기수 수 및 기수별 운영 기간 목록
  - 전체 참여자 수

### 참여자 관리

**참여자 목록 조회**
- 기수별 버튼 필터 / 이름·닉네임·전화번호 검색
- 컬럼: 이름 | 닉네임 | 전화번호 | 기수 | 등록일 | 챌린지 피처드 수 | 관리
- 활성 참여자만 목록에 노출 (탈락자는 제거되므로 상태 구분 없음)
- (자율창작 피처드는 개발 스펙 아웃 — 프로토타입에는 컬럼이 남아 있으나 실제 구현 제외)

**개별 추가**
- 닉네임 또는 전화번호로 앱 계정 검색 → 결과 드롭다운에서 선택 (2글자 이상 입력 시 노출)
- 선택 시 이름(실명) 자동 입력 — 실명 오타 등 케이스 대응을 위해 수정 가능하도록 함
- 실명은 운영 식별용으로만 사용하며 앱에는 표시되지 않음
- 기수 선택 후 추가

**CSV 일괄 추가**
- 형식: `전화번호, 기수` 2컬럼
- 업로드 후 미리보기 테이블 표시 (전화번호·이름·닉네임·기수·신규/덮어쓰기 여부)
- 중복 전화번호: 이름·닉네임·기수 덮어쓰기
- 계정 미조회 시: "계정 미확인" 상태로 등록

**탈락 처리**
- 목록에서 탈락 버튼 → 확인 모달 후 목록에서 즉시 제거
- 탈락 시 `is_tester = false`로 업데이트되어 앱 배지·권한 해제
- 별도 복귀 기능 없음 — 다시 참여시키려면 "참여자 추가"로 재등록

## 필요 API

| 메서드 | 경로 | 설명 |
|--------|------|------|
| GET | `/api/testers/cohorts` | 기수 목록 조회 |
| POST | `/api/testers/cohorts` | 기수 등록 |
| PATCH | `/api/testers/cohorts/:id` | 기수 수정 |
| GET | `/api/app-users?q=...` | 닉네임 또는 전화번호로 앱 계정 검색 (실명, 닉네임 반환) |
| GET | `/api/testers?cohort=n&status=active` | 참여자 목록 조회 |
| POST | `/api/testers` | 참여자 등록 |
| POST | `/api/testers/bulk` | CSV 일괄 등록 |
| DELETE | `/api/testers/:id` | 참여자 탈락 처리 (`is_tester = false` 업데이트 + 목록에서 제거) |

## Amplitude 연동

참여자 등록 / 탈락 처리 시 서버에서 Amplitude Identify API 직접 호출. 상세 내용 → `체험단_코호트_설정.md`

| User Property | 값 | 시점 |
|---------------|-----|------|
| `is_tester` | `true` | 참여자 등록 시 |
| `tester_cohort` | `1`, `2`… | 참여자 등록 시 |
| `is_tester` | `false` | 탈락 처리 시 (tester_cohort 유지) |

## 참고 사항

- (미완료) 닉네임·전화번호 통합 검색 API — 기존 앱 서버에서 제공 가능한지 확인 필요
- `is_tester` 변경 시 앱 내 배지 반영 시점 — 개발자 결정 사항
- (미완료) 체험단 배지 노출 영역 및 기간
  - **프로필**: 영구
  - **갤러리·댓글**: 기수 종료일 이후 30일, 자동 만료
  - 중도 참가자 포함 모두 기수 시작일·종료일 기준으로 계산. 개인 등록일 기준으로 조정 없음
- 체험단 계정 댓글 권한: `is_tester = true` 조건으로 제어
