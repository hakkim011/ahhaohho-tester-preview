설정 화면 — 아크크 작품집 만들기

Page ID : gallery
Screen : 설정

노출 조건 : is_tester = true인 경우에만 항목 노출
아이콘 : 작품집 관련 아이콘 (앨범/포트폴리오 계열)

① 기본 상태
- 설정 화면 하단 영역에 "아크크 작품집 만들기" 항목 노출
- 탭 시 전체 공개 게시물 유무 확인 API 호출

② 로딩 상태
- 버튼 우측에 스피너 표시, 항목 비활성화 (탭 불가)
- 로딩 중 설정 화면 딤(dim) 처리 → 다른 항목 탭 및 뒤로 가기 비활성화
- 공개 게시물 없음 → 토스트: "모든 사람에게 공개된 게시물이 있어야 작품집을 만들 수 있어요." + 딤/로딩 해제
- API 오류 → 토스트: "잠시 후 다시 시도해 주세요." + 딤 해제

③ 완료 상태 — OS 공유 시트
- URL 발급 완료 시 딤 해제 + iOS Share Sheet / Android Intent 호출
- 전달 값: `https://ahhaohho.com/portfolio/{userId}`
- 버튼 재탭 시 동일 URL 재사용 (새로 생성 안 함)

포함 게시물 범위
- 어드민에서 지정한 해당 유저의 체험단 기간 내 생성된 게시물만 포함
- 전체 공개 게시물만 포함 (나만보기 제외)
- 게시물 유형: 챌린지 게시물 + 자율 게시물 모두 포함
- 게시물 추가·비공개 전환·삭제는 URL 접속 시 실시간 반영 (최신 상태 기준)
- 체험단 기간 종료 후에도 URL 정상 접근 가능 — 기간 내 생성된 게시물의 최신 상태 열람 유지
- 탈락 처리(`is_tester = false`) 시 기발급 URL 포함 404 처리

챌린지 상세 화면 — 피처드 프로젝트 캐러셀

Page ID : gallery
Screen : 갤러리 (챌린지 상세)

챌린지 상세 화면 상단 이미지 영역에 피처드 프로젝트 캐러셀 추가
- 첫 번째 슬라이드: 기존 챌린지 썸네일 (고정)
- 이후 슬라이드: 피처드 풀 전체 중 최대 10개를 랜덤 순서로 노출 (피처드 지정 개수 제한 없음)
- 피처드가 없는 챌린지는 화살표 버튼 미노출, 기존 썸네일만 표시

① 화살표 버튼
- 첫 번째 슬라이드(썸네일)에서는 우측 화살표만 노출
- 한 번 넘기면 좌우 화살표 모두 노출
- 마지막 슬라이드에서는 좌측 화살표만 노출
- 탭 또는 스크롤 모두 슬라이드 이동 동작

② 메이커 닉네임 표시
- 피처드 슬라이드에 한해 이미지 상단에 "{닉네임}님의 프로젝트" 레이블 노출
- 챌린지 썸네일 슬라이드에서는 미노출

③ 프로젝트 이미지/영상
- 피처드 게시물의 첫 번째 미디어 노출 (이미지 또는 영상)
- 영상의 경우 자동 재생(loop), 사운드 기본 off (on 불가)

④ 캐시 정책
- 피처드 목록은 챌린지 상세화면 진입 시점 기준으로 로드
- 화면을 나갔다가 재진입 시 새로 조회 (세션 내 캐시 없음)
- 커뮤니티 외부·내부 동일하게 노출

⑤ 샛별 아이콘 (피처드 아이콘)
- 피처드 프로젝트임을 상징하는 아이콘 태그 노출 (앱 노출 명칭: 샛별)
- 노출 위치: 이미지·영상 우측 상단, 아이콘만 (텍스트 없음)
- 탭 시 별도 동작 없음
- 다수의 미디어 프로젝트일 경우 모든 페이지에 동일하게 노출
- 피처드 취소 시 태그 제거 타이밍: 화면을 나갔다가 재진입 시 새로 조회 기준과 동일 (세션 내 캐시 없음)

알림 — 챌린지 피처드 선정

Page ID : feed, notification
Screen : 갤러리 메인, 알림 보기
경로 : 피드 > 알림 보기

알림 구성 요소
- 1-1) 샛별 아이콘
- 1-2) 알림 내용: `{챌린지명} 챌린지의 샛별로 선정됐어요! 축하해요 🎉`
- 1-3) 프로젝트 썸네일: 해당 프로젝트의 게시물 썸네일 출력
- 1-4) 알림 발송 시점: 방금 전 (1시간 이내) / 1시간~23시간 / 1일~6일 / 1주~

알림 정책
- 피처드 취소 시 이미 발송된 알림에 영향 없음 (최초 발송 메시지 그대로 유지)
- 해당 화면 진입 후 이탈할 경우, 새로운 알림은 모두 읽음 상태로 전환 (모든 알림 공통)
- 알림 탭 시 이동 페이지: 해당 챌린지 상세 화면 진입 후 샛별 슬라이드로 바로 이동
  - 딥링크에 postId 포함 → 랜덤 노출 풀과 무관하게 해당 게시물 슬라이드를 첫 번째로 고정 노출

댓글 시트
갤러리 게시물 카드 탭 시 하단에서 올라오는 댓글 시트
조건 (진입) : 체험단 멤버(is_tester = true)의 게시물에 한해 댓글 시트 노출 — 비체험단 유저의 게시물에는 댓글 시트 없음
조건 (조회) : 모든 유저 — 체험단 멤버의 게시물이면 누구나 댓글 시트 열람 가능
조건 (작성) : is_tester = true인 경우에만 댓글 입력창 활성화 (기수 종료 후에도 is_tester 유지이므로 작성 가능)
배경 : 댓글 기능은 현재 커뮤니티 영역에서만 제공됨. 갤러리 게시물 댓글은 체험단 한정으로 운영하며, 추후 정식 댓글 기능 출시 시 별도 조건(친구 관계 기반 등)으로 대체될 수 있음

댓글 없음 안내 문구 (empty state)
댓글이 없을 때 출력되는 안내 문구
정책 : 커뮤니티 댓글 시트 empty state와 동일한 문구·디자인 사용

댓글 입력
하단 고정 영역에 댓글 작성 가능
정책
is_tester = false(탈락)인 경우, 체험단이 아닌 일반 회원과 동일하게 입력창 비활성화

댓글 좋아요
정책 : 커뮤니티 댓글 좋아요와 동일하게 동작. 좋아요 수신 시 알림 정책도 동일하게 적용

추천 리액션 미노출 ★
정책 : 체험단에서는 노출하지 않음. 동일 문구 댓글 도배로 미션을 손쉽게 완료하는 케이스를 방지하기 위함

체험단 배지 (tester_badge)
체험단 참여자 계정에 부여되는 배지
조건 : is_tester = true
배경 : 체험단 참여자임을 앱 내에서 시각적으로 구분하기 위해 배지를 부여. 갤러리·댓글에는 기수 종료일 이후 30일까지 노출되며, 프로필에는 영구 기록됨

공통 노출 위치 : 닉네임·레벨 우측
예외 처리 : 탈락(is_tester = false) 시 전 위치 즉시 비노출

만료 계산 기준
- 갤러리·댓글 배지 노출 만료(`오늘 ≤ end_date + 30일`) 판단은 서버 기준으로 처리
- 클라이언트는 서버 응답 값에 따라 노출 여부 결정

중도 케이스
- 중도 이탈 (탈락 처리): is_tester = false로 변경 → 배지 즉시 전 위치 비노출. 이후 복귀 불가.
- 중도 참가 (기수 도중 합류): 어드민에서 참여자 등록 시점부터 is_tester = true로 설정. 해당 기수 end_date 기준 동일하게 만료일 계산.

[노출화면 1] 갤러리 — 게시물 카드
작성자 닉네임·레벨 우측 노출
노출 조건 : is_tester = true AND 오늘 ≤ cohort[tester_cohort].end_date + 30일
정책 : 비활동 유저는 게시물 자체가 없으므로 별도 처리 불필요

[노출화면 2] 댓글 시트
댓글 작성자 닉네임·레벨 우측 노출
노출 조건 : is_tester = true AND 오늘 ≤ cohort[tester_cohort].end_date + 30일
예외 처리 : 기간 만료 시 기존 댓글에도 소급하여 비노출

[노출화면 3] 프로필
닉네임 우측 영구 노출
노출 조건 : is_tester = true (기간 제한 없음)
정책 : 체험단 참여 이력을 프로필에 영구 기록
