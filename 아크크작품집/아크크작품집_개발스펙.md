---

## 1. 개요

앱 내 창작 활동(공개 게시물)을 외부에서 열람할 수 있는 공유 가능한 웹 작품집.
체험단 수료 증빙 및 보호자·지인 공유 수단으로 활용. KPI 확인 후 앱 내 정식 기능으로 확장 여부 결정.

- 인터뷰 근거: 과학관 입문과정 이희준 인터뷰 (2026.05.22) — 부모가 발명반 포트폴리오 제출 시 날짜 뒤섞인 사진/영상을 수동으로 정리한 경험 언급
- 교사 니즈: 실물 문서(PDF)로 저장 가능한 수료 증빙 수단 요청 다수

> **접근 권한**: 체험단 참여자(`is_tester = true`) 전용. 반응이 좋을 경우 전체 유저 출시 예정.

---

## 2. 유저 플로우

### 2-1. 작품집 생성 및 공유 (앱)

```
설정 화면
└─ "아크크 작품집 만들기" 버튼 (is_tester = true인 경우에만 노출)
└─ 버튼 탭
   ├─ 버튼 비활성화 + 스피너 표시 (URL 발급 중)
   ├─ 공개 게시물 조회 (API)
   │   ├─ 게시물 없음 → 안내 토스트: "모든 사람에게 공개된 게시물이 있어야 작품집을 만들 수 있어요"  [버튼 활성화]
   │   └─ 게시물 있음 →
   │       고유 URL 발급 (없으면 신규 발급, 있으면 기존 URL 재사용)
   │       └─ OS 공유 시트 호출 (URL 포함)  [버튼 활성화]
   └─ API 오류 시 → 오류 토스트 + 버튼 활성화
```

> **로딩 UX 기준**: 프로그레스바 없음. 버튼 스피너만 적용. 수 초 이내 완료 예상.

### 2-2. 작품집 열람 (웹)

```
공유 URL 접속 (ahhaohho.com/portfolio/{userId})
└─ 서버: 해당 유저의 공개 게시물 조회
   ├─ 유저 미존재 또는 탈퇴 → 404 페이지 ("더 이상 존재하지 않는 작품집이에요")
   ├─ 공개 게시물 없음 → 빈 상태 페이지 안내
   └─ 정상 →
       표지 + 카드 그리드 렌더링
       카드 탭 → 상세 모달 (캐러셀)
       "PDF로 저장" 버튼 (웹/모바일 모두 동작)
```

## 3. URL 구조

| 항목 | 내용 |
|------|------|
| 형식 | `ahhaohho.com/portfolio/{userId}` |
| 발급 시점 | "아크크 작품집 만들기" 버튼 최초 탭 시 |
| 재사용 | 버튼 재탭 시 동일 URL 유지 (새로 생성 안 함) |
| 유효기간 | 영구 (계정 유지 시) |
| 탈퇴 시 | 404 처리 |
| 탈락 처리 시 (`is_tester = false`) | 404 처리 — 기발급 URL도 접근 불가 |
| 체험단 기간 종료 후 | 정상 접근 가능 — 기간 내 생성 게시물의 최신 상태 열람 유지 |
| 접근 권한 | 링크 보유자 누구나 열람 가능 (로그인 불필요) |

---

## 4. 데이터 아키텍처

작품집 URL 접속 시 서버가 해당 유저의 공개 게시물을 그 시점에 실시간으로 DB에서 조회해 렌더링. 게시물 추가·비공개 전환·삭제가 다음 접속 시 자동 반영.

---

## 5. 포함 게시물 기준

| 항목 | 기준 |
|------|------|
| 공개 범위 | **전체 공개** 게시물만 포함 |
| 게시물 유형 | 챌린지 게시물 + 자율 게시물 모두 포함 |
| 생성 기간 | 어드민에서 지정한 해당 유저의 체험단 기간 내 생성된 게시물만 포함 |
| 게시물 상태 | URL 발급 또는 접근 시점의 현재 상태 기준 (수정 내용 실시간 반영) |
| 정렬 | 게시 날짜 최신순 |
| 삭제된 게시물 | 실시간 제외 (URL 접속 시 최신 데이터 기준 렌더링) |

## 6. 화면 구성

### 6-1. 표지

> 디자인 변경 예정. 데이터 구성 기준만 명시.

| 요소 | 내용 |
|------|------|
| 닉네임 | 유저 닉네임 |
| 총 작품 수 | 포함된 게시물 수 |
| 기수 | 체험단 기수 (예: 1기) |
| 체험단 기간 | 어드민에서 지정한 기수 운영 기간 (예: 2026.08.08 ~ 2026.10.03) |

### 6-2. 카드 그리드

3컬럼 그리드 (700px 이하 2컬럼, 480px 이하 1컬럼).

| 요소 | 내용 |
|------|------|
| 썸네일 | 미디어 최대 4개를 그리드로 노출 |
| 썸네일 레이아웃 | 1장: 단일 전체 / 2장: 좌우 2분할 / 3장: 좌 1장 크게 + 우 상하 2장 / 4장: 2×2 |
| 미디어 수 뱃지 | 미디어 2개 이상일 때 우하단에 이미지 아이콘 + 전체 개수 표시 |
| 챌린지명 | 카드 상단에 표시. `null`이면 영역 미노출 |
| 본문 | 최대 2줄 말줄임 처리 |
| 정렬 | 최신순 |

> **썸네일 레이아웃 적용 기준**: 웹 화면과 PDF 출력 모두 동일하게 적용할 것. (현재 프로토타입에서는 3장/4장 레이아웃이 PDF 스타일에만 적용되어 있어 웹 화면에서 미반영 — 개발 시 수정 필요)

### 6-3. 카드 상세 (모달)

전체 화면(100vw × 100vh), 좌 62%(미디어 캐러셀) + 우 38%(텍스트) 가로 분할.
모바일(700px 이하)에서는 상(55vh 미디어) + 하(텍스트) 세로 스택으로 전환.

| 요소 | 내용 |
|------|------|
| 미디어 캐러셀 | 이미지/영상 순서 유지. 좌우 화살표 + dot 인디케이터 표시 |
| 캐러셀 화살표 | 첫 슬라이드에서 이전 비활성, 마지막에서 다음 비활성 |
| 영상 | 인라인 재생, 사운드 기본값 off (`muted`), 자동 재생 없음 — 사용자가 수동 재생 |
| 모달 닫기 | 닫기 버튼 또는 배경(오버레이) 클릭. ESC 키 지원 |
| 모달 닫힐 때 | 재생 중인 영상 정지 및 currentTime 초기화 |
| 챌린지명 | `null`이면 영역 미노출 |
| 본문 | 최대 500자 |
| 날짜 | 게시 날짜 (YYYY.MM.DD) |
| 키보드 단축키 | ESC: 모달 닫기 / ←→: 캐러셀 탐색 |

### 6-4. PDF 저장

- 웹/모바일 구분 없이 모두 동작 (`window.print()`)
- 브라우저 인쇄 다이얼로그에서 "PDF로 저장" 선택 시 다운로드
- 출력 레이아웃: A4 가로, 표지 1장 + 작품 카드 1장/페이지
- 영상: **썸네일 이미지로 대체 출력** → **썸네일 URL 서버 제공 필요** (현재 프로토타입에서 미구현 — 개발 시 반영 필요)
- PDF 저장 버튼: 우하단 fixed 위치, 인쇄 시 숨김 처리
- 인쇄 시 숨김 처리 요소: 앱스토어 CTA, 섹션 헤더, PDF 저장 버튼, 모달

## 7. API 명세

### 7-1. 작품집 URL 발급 (앱 → 서버)

```
POST /api/portfolio/generate
Authorization: Bearer {token}

Response
{
  "url": "https://ahhaohho.com/portfolio/{userId}"
}
```

- 이미 발급된 경우 동일 URL 반환 (멱등성 보장)

---

### 7-2. 작품집 데이터 조회 (웹 → 서버)

```
GET /api/portfolio/{userId}

Response
{
  "user": {
    "name": "이희준",
    "total": 9,
    "cohort": 1,
    "cohort_period": {
      "start": "2026.08.08",
      "end": "2026.10.03"
    }
  },
  "posts": [
    {
      "title": "화산 폭발 만들기",   // 챌린지명. 없으면 null
      "description": "식초랑 베이킹소다를 섞었더니...",
      "created_at": "2025.05.16",
      "media": [
        {
          "type": "image",
          "url": "https://...",
          "thumbnail_url": null       // 이미지는 null 허용
        },
        {
          "type": "video",
          "url": "https://...",
          "thumbnail_url": "https://..." // 영상은 필수
        }
      ]
    }
  ]
}
```

**에러 응답**

| 상태코드 | 사유 | 프론트 처리 |
|---------|------|------------|
| 404 | 유저 미존재 or 탈퇴 | 404 안내 페이지 |
| 200 + posts: [] | 공개 게시물 없음 | 빈 상태 안내 페이지 |

## 8. 엣지 케이스

| 케이스 | 처리 |
|--------|------|
| 게시물 없는 상태에서 버튼 탭 | "모든 사람에게 공개된 게시물이 있어야 작품집을 만들 수 있어요" 토스트, URL 미발급 |
| 작품집 생성 후 게시물 전체 비공개 전환 | 웹 접속 시 빈 상태 안내 (URL은 유지) |
| 미디어 로드 실패 | 회색 플레이스홀더 표시 |
| 영상 thumbnail_url 누락 | 영상 아이콘 플레이스홀더 표시 |
| **탈락자 작품집 URL 접근** | 404 처리 — `is_tester = false` 시 기발급 URL도 404로 응답 |
| **체험단 기간 종료 후 URL 접근** | 정상 접근 가능 — 체험단 기간 내 생성된 게시물의 최신 상태를 계속 열람할 수 있음. 기간 이후 새로 생성된 게시물은 미포함 |

### 8-1. 앱스토어 다운로드 버튼

작품집 하단(푸터 바로 위)에 앱 설치 유도 CTA 섹션 고정 노출.

| 항목 | 내용 |
|------|------|
| 위치 | 카드 그리드 하단 |
| 문구 | "**아하오호**에서 나만의 창작 기록 남기기" ("아하오호" 네온 색상 강조) |
| 버튼 | App Store (iOS), Google Play (Android) — 각각 스토어 링크로 이동 |
| 노출 범위 | 모든 작품집 페이지 (작품 있음 / 빈 상태 모두) |
| 인쇄/PDF | `display: none` 처리 (인쇄 출력물 미포함) |

## 9. 개발 시 주의 사항

프로토타입(HTML)과 실제 구현 시 차이가 발생할 수 있는 항목.

| 항목 | 내용 |
|------|------|
| 영상 자동재생 | 스펙상 자동재생 없음 (사용자 수동 재생). muted 속성은 적용할 것 |
| 영상 썸네일 PDF 대체 출력 | 서버에서 thumbnail_url 필수 제공. PDF 출력 시 video 태그 대신 썸네일 img로 교체 필요 |
| 카드 그리드 3장/4장 레이아웃 | 웹 화면에도 동일하게 적용 필요 (현재 프로토타입은 PDF 스타일에만 구현됨) |
| footer 인쇄 숨김 | 인쇄/PDF 출력 시 footer 영역도 `display: none` 처리 필요 |

## 12. 참고

- 웹 뷰어 프로토타입 및 빈 상태·404 페이지 참고 파일은 GitHub 레포 내 `아크크/아크크작품집/` 폴더에서 확인
- 아크크 작품집은 앱 내 화면이 아닌 **외부 웹 링크(브라우저에서 열리는 웹 페이지)**이므로, React Native 컴포넌트로 재작성 불필요. 프로토타입 HTML/CSS/JS를 구현 참고용으로 활용하되, 반응형 대응 및 실제 API 연결은 별도 개발 필요.
